# 🎧 M3U Playlist Proxy

A lightweight proxy server for streaming and modifying M3U playlists on the fly. Perfect for IPTV users who want to filter, clean, or host their playlists with ease.

---

## 🚀 Features

- 🌐 Serve M3U playlists via HTTP
- 🧹 Filter or modify channels (rename, remove, or reorder)
- 🔄 Auto-refresh support for remote playlists
- 🔒 Optional IP whitelisting or token auth
- 📦 Lightweight and easy to deploy

---

## 🛠️ Customization

- Modify channel names
- Filter groups
- Inject custom headers
- Add EPG links

## 🌍 Vercel Deployment Guide

This guide explains how to deploy your own live version of the app on **Vercel**.

---

### ✅ Requirements

Before you begin, ensure you have the following:

* A **GitHub** account
* A **Vercel** account (sign up at [vercel.com](https://vercel.com))
* **Node.js** and **npm** installed (for local development, optional)

---

### 1️⃣ Fork or Clone the Repository

Start by obtaining a copy of the project:

* **Fork the repository** to your own GitHub account
      [Fork M3U-playlist-porxy](https://github.com/DisabledAbel/M3U-playlist-porxy/fork)
  **or**
* **Clone the repository** locally

```bash
git clone https://github.com/DisabledAbel/M3U-playlist-porxy.git
```

Push the code to a new repository in your own GitHub account if needed.

---

### 2️⃣ Deploy the Project to Vercel

1. Visit [vercel.com/new](https://vercel.com/new)
2. Select **“Import Git Repository”**
3. Choose your repository
4. Follow the prompts and click **“Deploy”**

Vercel will automatically build and deploy the project.

---

### 3️⃣ Configure Environment Variables (If Required)

If the application requires environment variables:

1. Open your Vercel project’s **Settings → Environment Variables**
2. Add the necessary variables and their values
3. Click **“Redeploy”** to apply the changes

> *Note:* If no environment variables are required, this step can be skipped.

---

### 4️⃣ Continuous Deployment

Whenever I push updates to your GitHub repository,
**Vercel automatically redeploys** the app to reflect the latest changes.

---

Your application is now live and fully deployed.
